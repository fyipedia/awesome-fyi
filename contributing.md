# Contribution Guidelines

Thank you for considering contributing to awesome-fyi!

## Adding an Entry

Please ensure your pull request adheres to the following guidelines:

- Search previous suggestions before making a new one to avoid duplicates.
- Make an individual pull request for each suggestion.
- Use the following format: `- [Name](URL) - Description.`
- Descriptions should be concise, clear, and end with a period.
- New categories or improvements to the existing categorization are welcome.
- Keep descriptions short and simple, but descriptive.
- Start the description with a capital letter.
- Check your spelling and grammar.
- The pull request should have a useful title and include a link to the package or resource and why it should be included.

## Updating an Entry

- If a package has been renamed, moved, or deprecated, please submit a PR to update or remove it.
- If a "Coming Soon" item has been released, please submit a PR to add the proper link.

## Quality Standards

To be included in this list, a project should meet these criteria:

- Be related to the FYIPedia ecosystem or compatible with FYI packages.
- Be actively maintained or in a stable state.
- Have clear documentation or a README.
- Be open source with a recognized license.

Thank you for your suggestions!
